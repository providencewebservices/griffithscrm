# Griffiths CRM - Development Guidelines

## Package Manager

This project uses **bun**, not npm or pnpm. Always use `bun` for running scripts:

```bash
bun run dev          # Start development servers
bun run build        # Build all packages
bun run build:web    # Build web app
bun run build:api    # Build API
bun run db:push      # Push database schema changes (development only)
bun run db:generate  # Generate SQL migration files
```

## Database

### Development vs Production

- **Development**: Use `bun run db:push` to apply schema changes directly
- **Production**: Uses SQL migration files generated by `bun run db:generate`

### Schema Location

Database schema is defined in `packages/shared/src/db/schema.ts`. After making changes:

1. Run `bun run db:generate` to create migration files
2. Migration files go to `packages/shared/drizzle/`
3. Migrations run automatically on production container startup

### Production Migrations

Production uses programmatic migrations via `drizzle-orm/postgres-js/migrator` (NOT `drizzle-kit` CLI). The migration script is at `apps/api/scripts/migrate.ts` and runs via `docker-entrypoint.sh`.

**Important**: postgres.js requires `{ max: 1 }` connection pool for migrations.

## Docker & Deployment

### Bun in Docker

- Use `oven/bun:1.3.4-alpine` or newer (older versions have lockfile compatibility issues)
- Do NOT use `--frozen-lockfile` - known bug with bun in Docker/monorepos ([bun#12252](https://github.com/oven-sh/bun/issues/12252))
- Dependencies must be hoisted to root `package.json` for Docker builds (drizzle-orm, postgres)

### Deploying Changes

1. **API changes**: Build Docker image, push to ECR, force ECS deployment
2. **Frontend changes**: Build with `bun run build`, sync to S3, invalidate CloudFront
3. **Schema changes**: Generate migrations, rebuild Docker image, deploy

### AWS Region

- Primary region: `eu-west-2` (London)
- CloudFront/ACM certificates: `us-east-1` (required by CloudFront)

## Terraform

Infrastructure is in `terraform/` directory. Key files:

- `ecs.tf` - ECS cluster, service, task definition
- `rds.tf` - PostgreSQL database and DATABASE_URL in SSM
- `cloudfront.tf` - S3 bucket and CloudFront distribution
- `terraform.tfvars` - Environment-specific variables

### Common Terraform Tasks

```bash
cd terraform
terraform plan    # Preview changes
terraform apply   # Apply changes
```

### RDS SSL

RDS requires SSL connections. The `DATABASE_URL` in SSM includes `?sslmode=require`.

### CloudFront Custom Domain

If using external DNS (not Route53), set `web_domain` in tfvars but leave `route53_zone_id` empty. The ACM certificate must be manually validated via DNS.

## UI Patterns

### List/Table Views

Tables showing entities should follow this pattern:

- **View Button**: Use a simple "View" button in the actions column, not a dropdown menu with ellipsis
- **No Edit/Delete on Lists**: Edit and delete actions belong on the detail page only
- **Add Button**: List pages can have an "Add" button in the header for creating new items
- **Navigation Focus**: List views are for browsing and navigating to detail pages

```tsx
// Good - Simple View button
<TableCell>
  <Link to={`/app/categories/${item.id}`}>
    <Button variant="ghost" size="sm">View</Button>
  </Link>
</TableCell>

// Bad - Dropdown with multiple actions
<TableCell>
  <DropdownMenu>
    <DropdownMenuTrigger>
      <MoreHorizontal />
    </DropdownMenuTrigger>
    <DropdownMenuContent>
      <DropdownMenuItem>View</DropdownMenuItem>
      <DropdownMenuItem>Edit</DropdownMenuItem>
      <DropdownMenuItem>Delete</DropdownMenuItem>
    </DropdownMenuContent>
  </DropdownMenu>
</TableCell>
```

### Detail Pages

Detail pages handle all CRUD operations for a single entity:

- **Edit**: Button in header or inline editing
- **Delete/Archive**: In actions dropdown or dedicated button
- **Related Data**: Show and manage related entities (e.g., products in a category)

## Troubleshooting

### ECS Logs

```bash
aws logs tail /ecs/griffiths-crm-prod --follow --region eu-west-2
```

### ECS Service Status

```bash
aws ecs describe-services --cluster griffiths-crm-prod --services griffiths-crm-prod-api --region eu-west-2
```

### ECS Exec (Shell Access)

```bash
aws ecs execute-command --cluster griffiths-crm-prod --task <task-id> --container api --interactive --command "/bin/sh" --region eu-west-2
```

### Docker Build Issues

If builds fail with stale errors, use `docker build --no-cache`
